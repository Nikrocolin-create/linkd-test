?Есть ли готовые решения, чтобы возвращать каждый раз следующее число (counter), мб какой нибудь микросервис
nextval() в PostgreSQL — атомарная операция на уровне сервера, вне рамок транзакции

Redis INCR — самое распространённое решение:
Long id = redisTemplate.opsForValue().increment("link:id:counter");

IDENTITY — делегирует генерацию БД (GENERATED BY DEFAULT AS IDENTITY). Hibernate просто делает INSERT и читает сгенерированный ID обратно.
SEQUENCE — Hibernate сам управляет последовательностью: перед INSERT делает отдельный запрос SELECT nextval('links_seq'), получает ID заранее, и только потом INSERT.

Колонка объявлена как GENERATED BY DEFAULT AS IDENTITY — это значит, что явная вставка своего id разрешена (в отличие от GENERATED ALWAYS, где PostgreSQL запрещал бы это без OVERRIDING SYSTEM VALUE).
Под капотом PostgreSQL уже создал последовательность links_id_seq. Hibernate с GenerationType.SEQUENCE будет вызывать nextval('links_id_seq') — именно то, что нужно.
